# Управление Швейной Фабрикой - Демо-версия

**Корпоративная платформа для управления сетью швейных фабрик.**

## 🚀 Быстрый старт

### Требования
- Node.js 18+ установлен
- npm или yarn пакетный менеджер

### Установка

1. Установите зависимости:
```bash
npm install
```

2. Запустите сервер разработки:
```bash
npm run dev
```

3. Откройте браузер:
Перейдите по ссылке [http://localhost:3000](http://localhost:3000)

### Сборка для продакшена

```bash
npm run build
npm start
```

## 🔐 Данные для входа

**Логин:** `admin`  
**Пароль:** `admin123`

Внимание: Для входа перейдите на страницу [http://localhost:3000/login](http://localhost:3000/login)

## 📋 Возможности

### Дашборд
- Обзор всех фабрик в реальном времени
- Ключевые метрики: выручка, прибыль, эффективность
- Графики производства и прибыли
- Последние уведомления
- Сравнение производительности фабрик

### Производство
- Отслеживание заказов с визуализацией прогресса
- Мониторинг производственных партий
- Обзор мощностей фабрик
- Управление статусами заказов (в ожидании, в работе, завершен, задержан)
- Отслеживание уровня брака по партиям

### Сотрудники
- Справочник сотрудников с показателями эффективности
- Отслеживание эффективности и визуализация
- Автоматический расчет заработной платы
- Начисление бонусов на основе KPI
- Рейтинг лучших сотрудников
- Фильтрация и сортировка по фабрикам

### Финансы
- Финансовые метрики в реальном времени
- Отслеживание выручки и расходов
- Анализ рентабельности
- Управление денежными потоками
- История транзакций
- Финансовая производительность фабрик
- Просмотр по периодам (сегодня/месяц)

### Аналитика
- Анализ трендов производства
- Мониторинг уровня брака
- Сравнение эффективности фабрик
- Распределение приоритетов заказов
- Рейтинг лучших сотрудников
- Общий показатель операционного здоровья

## 🎨 Технологический стек

- **Frontend:** Next.js 14 с App Router
- **Language:** TypeScript
- **Styling:** Tailwind CSS
- **Charts:** Chart.js + react-chartjs-2
- **Icons:** Lucide React
- **Data:** Mock данные (готовы для интеграции с API)

## 📊 Mock данные

Демо использует комплексные mock данные, представляющие:
- 5 фабрик с показателями производства
- 5 заказов с разными статусами
- 4 активных производственных партии
- 6 сотрудников с данными эффективности
- Финансовые транзакции и история
- Аналитика и данные уведомлений

Все mock данные находятся в `data/mockData.ts`

## 🐛 Известные ограничения (Демо)

- Данные статического mock (без подключения к backend)
- Нет обновлений в реальном времени
- Нет аутентификации (только демо)
- Один пользователь
- Ограниченная интерактивность
